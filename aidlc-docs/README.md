## Documentation Controls

- Update the active artifact instead of creating `_new`, `_revised`, `_modified`, or similarly duplicated files.
- Keep one requirements document, one workflow plan, one code-generation plan, and the mandatory Build and Test artifacts per active change.
- Put implementation details in application code and tests; documentation records decisions, scope, and verification evidence.
- Mark abandoned plans with a superseded banner and stop extending them.
- Keep `aidlc-state.md` focused on the active workflow plus a short historical summary.
- Keep `audit.md` append-only and complete; it is the required exception to the concision policy.
- Create additional design artifacts only when an approved stage requires them or unresolved behavior cannot be expressed safely in the active plan.
